Etapa 1: Criação da Solução e do Projeto de API

Nesta primeira etapa, criamos a estrutura básica da nossa solução e do projeto de API mínima usando o .NET CLI.

Passos Executados

1. Criação da Solução

Primeiro, criamos um arquivo de solução (solution) chamado CadClientes.sln para agrupar nossos projetos. O comando utilizado foi:

dotnet new sln -n CadClientes

2. Criação do Projeto de API

Em seguida, criamos o projeto de API mínima com o nome ClientesAPI. Este projeto já vem com uma estrutura básica para uma API web.

dotnet new webapi -n ClientesAPI

3. Adicionando o Projeto à Solução

Para vincular o projeto à solução, executamos o comando a seguir:

dotnet sln CadClientes.sln add ClientesAPI/ClientesAPI.csproj

4. Limpeza do Projeto

O modelo padrão de API mínima do .NET inclui um endpoint de exemplo (/weatherforecast). Para começar com um projeto limpo, removemos esse código do arquivo Program.cs.

Conteúdo Original de Program.cs


var builder = WebApplication.CreateBuilder(args);

// Add services to the container.
// Learn more about configuring OpenAPI at https://aka.ms/aspnet/openapi
builder.Services.AddOpenApi();

var app = builder.Build();

// Configure the HTTP request pipeline.
if (app.Environment.IsDevelopment())
{
    app.MapOpenApi();
}

app.UseHttpsRedirection();

var summaries = new[]
{
    "Freezing", "Bracing", "Chilly", "Cool", "Mild", "Warm", "Balmy", "Hot", "Sweltering", "Scorching"
};

app.MapGet("/weatherforecast", () =>
{
    var forecast =  Enumerable.Range(1, 5).Select(index =>
        new WeatherForecast
        (
            DateOnly.FromDateTime(DateTime.Now.AddDays(index)),
            Random.Shared.Next(-20, 55),
            summaries[Random.Shared.Next(summaries.Length)]
        ))
        .ToArray();
    return forecast;
})
.WithName("GetWeatherForecast");

app.Run();

record WeatherForecast(DateOnly Date, int TemperatureC, string? Summary)
{
    public int TemperatureF => 32 + (int)(TemperatureC / 0.5556);
}
