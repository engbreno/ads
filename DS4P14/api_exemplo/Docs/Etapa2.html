Etapa 2: Criando uma Rota de Teste e Conceitos de API RESTful

Nesta etapa, vamos adicionar uma rota de teste (endpoint) à nossa API para verificar se tudo está funcionando e para retornar informações básicas sobre o projeto. Além disso, vamos explorar os conceitos fundamentais de APIs RESTful.

O que é uma API RESTful?

Uma API (Application Programming Interface) é um conjunto de regras e definições que permite que diferentes aplicações se comuniquem entre si. REST (Representational State Transfer) é um estilo de arquitetura de software que define um conjunto de restrições para a criação de APIs.

Uma API que segue os princípios do REST é chamada de RESTful. Suas principais características são:

  • Arquitetura Cliente-Servidor: A interface do usuário (cliente) é separada do armazenamento de dados (servidor), permitindo que evoluam de forma independente.
  • Stateless (Sem Estado): Cada requisição do cliente para o servidor deve conter toda a informação necessária para ser compreendida e processada. O servidor não armazena nenhum estado sobre o cliente entre as requisições.
  • Cacheable: As respostas do servidor devem, implicitamente ou explicitamente, definir-se como cacheáveis ou não, para que o cliente possa reutilizar respostas para requisições futuras, melhorando a performance.
  • Interface Uniforme: Esta é a restrição fundamental do REST e é composta por quatro princípios:
    • Identificação de Recursos: Os recursos (por exemplo, um cliente, um produto) são identificados por URIs (Uniform Resource Identifiers), como /clientes/123.
    • Manipulação de Recursos Através de Representações: O cliente interage com uma representação do recurso (geralmente em formato JSON ou XML), não com o recurso em si.
    • Mensagens Autodescritivas: Cada mensagem inclui informação suficiente para descrever como processá-la (ex: usando Media Types como application/json).
    • HATEOAS (Hypermedia as the Engine of Application State): As respostas do servidor devem conter links que guiem o cliente para as próximas ações possíveis.
  • Sistema em Camadas: Um cliente não consegue dizer se está conectado diretamente ao servidor final ou a um intermediário ao longo do caminho.

Em resumo, APIs RESTful usam os métodos do protocolo HTTP (GET, POST, PUT, DELETE, etc.) para operar sobre os recursos, tornando a comunicação padronizada e previsível.

Passos para Criar a Rota de Teste

1. Adicionando um Endpoint GET

Para criar nossa rota de teste, editamos o arquivo Program.cs e adicionamos um novo endpoint usando o método MapGet(). Este método mapeia requisições HTTP GET para um determinado caminho (URI).

Nossa rota será /info e retornará um objeto anônimo com informações do projeto em formato JSON.

Conteúdo Final de Program.cs


var builder = WebApplication.CreateBuilder(args);

// Add services to the container.
// Learn more about configuring OpenAPI at https://aka.ms/aspnet/openapi
builder.Services.AddOpenApi();

var app = builder.Build();

// Configure the HTTP request pipeline.
if (app.Environment.IsDevelopment())
{
    app.MapOpenApi();
}

app.UseHttpsRedirection();

app.MapGet("/info", () => new {
    Solucao = "CadClientes",
    Projeto = "ClientesAPI",
    VersaoDotNet = "9.0.304",
    Autor = "Alunos",
    Data = DateTime.Now.ToString("yyyy-MM-dd")
});

app.Run();

2. Testando a Rota

Com a aplicação em execução, você pode acessar a URL https://localhost:<porta>/info em seu navegador ou em uma ferramenta como o Postman. A resposta deverá ser um JSON com as informações que definimos.
